# Screenshot Checklist

Use this when collecting evidence for `design-review`.

## Required Viewports

- 1440 x 1000: main desktop composition.
- 1024 x 900: constrained desktop/tablet edge.
- 390 x 844: common mobile viewport.
- 1440 x 700: short laptop viewport.

## Capture

- First viewport.
- Middle proof/mechanism section.
- CTA section.
- Footer.
- Any interactive or motion-heavy state.
- Mobile menu if navigation changes.

## Inspect

- Does the first viewport communicate product/category/value/CTA?
- Is there a hint of the next section?
- Does any text collide, overflow, or become too small?
- Are product visuals legible at mobile size?
- Are line lengths and heading breaks intentional?
- Does the layout feel designed on mobile, not just stacked?
- Are focus/hover states stable and visible?
- Are images sharp, correctly cropped, and not stock-like filler?
- Is there horizontal overflow?
- Does reduced motion preserve meaning?
